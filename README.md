Feature Engineering


კატეგორიული ცვლადების რიცხვითად გადაქცევა
მონაცემთა ნაკრები შეიცავს მრავალ კატეგორიულ მახასიათებელს (მაგ., სამეზობლო, სახლის სტილი, მდგომარეობა), მათ პირდაპირ ვერ გამოვიყენებთ.
ამიტომ გავაკეთეთ label encoding  ფაქტორიზაციით:
X_cleaned[col] = pd.factorize(X_cleaned[col])[0]
ეს მიდგომა მახასიათებლის ფარგლებში თითოეულ კატეგორიას უნიკალურ მთელ რიცხვს ანიჭებს.
უპირატესობაა - მარტივი და სწრაფი გამოთვლა და კარგად მუშაობა ხეზე დაფუძნებულ მოდელებთან(მაგ. Random Forest)

NaN-ების დაჰენდლვა
NaN-ეიბს დაჰენდვლა მნიშვნელოვანია რადგან ხდის დატასეტს მთლიანად გამოყენებადს
  რიცხვითი მნიშვნელობები:
  რიცხვით სვეტებში დაკარგული მნიშვნელობები შეიცვალა თითოეული სვეტის მედიანით
  X_cleaned[num_cols] = X_cleaned[num_cols].fillna(X_cleaned[num_cols].median())
  მედიანით განაწილება უფრო ნაკლებად იცვლება ვიდრე საშუალოთი.

  კატეგოირული მნიშვნელობები:
  კატეგორიულ სვეტებში დაკარგული მნიშვნელობები შეიცვალა სტრინგით "None":
  X_cleaned[col] = X_cleaned[col].fillna("None")

მიდგომები ქლინინგისთვის:
წაიშალა რამდენიმე სვეტი, რომელშიც ბევრი დაკარგული მნიშვნელობები იყო.
მათი გათვალისწინება ხმაურს გამოიწვევდა, ამოღებით სანდოობა გავაუმჯობესეთ.




Feature Selection

გამოყენებული მეთოდი: Recursive Feature Elimination (RFE)
RFE მუშაობს შემდეგი გზებით:
   მოდელის თრეინინგი მახასიათებლების სრულ კომპლექტზე
   მახასიათებლების დარანკვა მნიშვნელობის მიხედვით
   ვშლით ყველაზე ნაკლებად მნიშვნელოვან მახასიათებლებს
   ვიმეორებთ პროცესს სანამ მახასიათებლების სასურველი რაოდენობა არ დაგვრჩება (ამ შემთხვევაში 30) 
ეს მეთოდი უფრო მარტივ მეთოდებთან შედარებით უკეთ ფილტრავს ხმაურს.

საერთო ჯამში, Feature Selection-მა მნიშვნელოვანი როლი ითამაშა უფრო სტაბილური და ეფექტური მოდელის შექმნაში.





Training
გამოყენებული მოდელები:
1. Linear Regression (Baseline)
Train RMSE: 0.16353
Validation RMSE: 0.20709
Gap: 0.04356

Linear Regression გამოვიყენეთ baseline მოდელად რათა გვეჩვენა როგორ მუშაობს მარტივი წრფივი მიდგომა დატასეტზე.
ის მახასიათებლებსა და სამიზნეს შორის წრფივ კავშირს იყენებს, რაც ძალიან შემზღუდველია. 
გვაქვს underfitting, რაც ნიშნავს, რომ მოდელი ძალიან მარტივია კომპლექსური შემთხვევების აღსაწერად.


2. Decision Tree
Train RMSE: 0.05880
Validation RMSE: 0.20709
Gap: 0.14830

მონაცემებში არაწრფივი დამოკიდებულებების აღსაწერად გამოიცადა გადაწყვეტილების ხე.
ის ძალიან კარგად ერგება სასწავლო მონაცემებს, მაგრამ ვერ ახდენს უნახავ მონაცემებზე კარგად განზოგადებას.
გვაქვს overfitting.


3. Lasso Regression
Train RMSE: 0.18960
Validation RMSE: 0.25079
Gap: 0.06119

ლასოს რეგრესია გამოყენებული იქნა რეგულარიზაციის დასანერგად და მოდელის სირთულის შესამცირებლად. ის აჯარიმებს დიდ კოეფიციენტებს და ეფექტურად არჩევს მახასიათებლებს.
თუმცა, ამ შემთხვევაში underfitting გვაქვს, რაც იმაზე მიუთითებს, რომ მოდელი ძალიან შეზღუდული გახდა.

4. Random Forest (ცდა 1)
Parameters:
n_estimators = 700
max_depth = None
bootstrap = False
Results:
Train RMSE: 0.04771
Validation RMSE: 0.19866
Gap: 0.15095

ვცადეთ რენდომ ფორესტი მაგრამ ღრმა ხეების კონფიგურაციით და ბუტსტრეპინგის გარეშე თითქმის იდეალურად ისწავლა თრეინინგ დატა.
ამიტომ არ გვაქვს კარგი განზოგადება ვალიდაციაზე და ვიღებთ ექსტრემალურ overfitting-ს.


5. Tuned Random Forest (რენდომ ფორესტის მეორე ცდა)
Parameters:
n_estimators = 500
max_depth = 15
min_samples_split = 5
Results:
Train RMSE: 0.06676
Validation RMSE: 0.15229
Gap: 0.08553

ამ მოდელს ფრთხილად შევურჩიეთ არგუმენტები, რათა მიღწეულიყო ბალანსი bias-სა და variance-ს შორის. ხის სიღრმის შეზღუდვა და სფლითების კონტროლი გვარიდებს overfitting-ს.
ვალიდაციაზე გვაქვს ყველაზე ნაკლები ერორი ამ მოდელზე, ამ ყველაფრის გამო საბოლოოდ მოცემულ მოდელს ვირჩევთ.
